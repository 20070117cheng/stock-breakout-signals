# 大漲訊號自動選股系統 — 設計文件

日期：2026-07-03
依據：林則行《大漲的訊號》（大是文化）＋使用者需求訪談

## 目標

把書中「創新高價股」獲利公式做成全自動雲端系統：

- 每日掃描台股（上市＋上櫃全部，約 1800 檔）與美股（S&P 500 + NASDAQ 100，約 600 檔）
- 產生買進候選（9 項檢核表評分卡）與持股賣出警報（停損 / 基本面 / 賣壓比例）
- Gmail 通知買賣訊號
- GitHub Pages 新手友善儀表板
- 使用者電腦關機也能運作；不依賴任何 AI 訂閱；全免費

### 賣出三條件（第四章）

1. **停損**：收盤價 ≤ 買價 × 0.92（書 p.215-228：8%，可設 7~10%）→ 立即賣出（紅色警報）
   - 待補：書 p.227-228「跌破〇日來最低價」輔助條件，PDF 缺字，等使用者提供照片
2. **基本面惡化**：最新一季稅前淨利 YoY < 20% → 賣出（書 p.192-193）
3. **技術面**：賣壓比例 SPR（20 營業日）≥ 117%（書 p.201-215，可設 116~118%）→ 賣出訊號（可觀察再賣，書 p.211-213）

SPR 演算法（書 p.203-208）：以每日 OHLC 將當日行進拆為買盤/賣壓段，按漲跌幅比例分配成交量為買進股數/賣出股數，SPR = 20 日賣出股數總和 ÷ 買進股數總和。書中範例（910/915/910/925/920，量12萬 → 買8萬賣4萬；920/925/930/905/910，量8萬 → 買3萬賣5萬；SPR=9/11=82%）作為單元測試基準。

### 資金管理

- 單檔 ≤ 總資產 10%（config `total_capital`，Email 附建議張數）
- 行情弱（⑨紅燈）時提示減量或不進場

## 架構

```
GitHub Repo（私人）
├── engine/               Python 選股引擎（無 AI 依賴）
│   ├── universe.py       成分股清單（Wikipedia S&P500/NDX、FinMind 台股清單）
│   ├── datastore.py      價格快取（parquet，增量更新）
│   ├── spr.py            賣壓比例
│   ├── signals/
│   │   ├── new_high.py   ①② 突破偵測、反彈幅度、平穩期
│   │   ├── fundamentals.py ③④⑤⑥⑧（僅對通過價格篩選的候選查詢，省 API）
│   │   ├── market.py     ⑨ 創新高數量比、前50大、紅綠燈
│   │   └── sell.py       停損/基本面/SPR 持股監控
│   ├── scoring.py        檢核表評分卡
│   ├── notify.py         Gmail SMTP（應用程式密碼）
│   ├── report.py         產生 docs/index.html 儀表板
│   └── run_daily.py      入口：--market tw|us
├── holdings.csv          使用者持股（GitHub 網頁直接編輯）
├── config.yaml           門檻、資產總額、Email
├── data/*.parquet        價格快取（workflow 自動 commit 回 repo）
├── docs/index.html       GitHub Pages 儀表板
└── .github/workflows/
    ├── tw.yml            台北 16:30（UTC 08:30）週一~五
    └── us.yml            台北 06:00（UTC 22:00）週二~六
```

資料源：價格全部走 yfinance（台股 .TW/.TWO 尾碼、美股原代號，批次下載）；台股財報/月營收走 FinMind 免費 API（只查候選股，避開限流）；美股財報走 yfinance per-ticker。

## 誠實限制（已告知使用者）

- ⑦ 與平穩期判定書中明言無法公式化 → 系統給近似分數＋人工確認清單
- 停損以每日收盤檢查，非盤中即時
- 美股長期獲利史僅 4 年（yfinance 限制），③ 打分依可得年數
- 方法在空頭勝率低（書 p.86：約 20%），⑨ 燈號控制曝險但不保證獲利
- GitHub Actions 私人 repo 每月 2000 分鐘免費額度；引擎設計目標單次 <15 分鐘

## 測試策略（TDD）

- SPR：書中數值範例精確重現
- 反彈幅度：王將 48%（p.231）、愛捷是 64%（p.70）重現
- 突破偵測、停損、季成長篩選：合成資料單元測試
- 整合：50 檔小樣本端對端跑通，產出儀表板與 Email 乾跑

## 實作順序

1. SPR + 反彈幅度 + 突破（核心數學，先測後寫）
2. datastore + universe（真實資料小樣本驗證）
3. fundamentals + scoring + market
4. sell + notify + report（儀表板）
5. run_daily + workflows + SETUP.md 新手教學
6. 全樣本試跑、驗證、交付
